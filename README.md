#  AHJ-Chat: Корпоративный Чат на WebSockets
[![Netology AHJ](https://github.com/Milka79rus/ahj-chat/actions/workflows/web.yml/badge.svg)](https://github.com/Milka79rus/ahj-chat/actions/workflows/web.yml)

Интерактивный клиент для корпоративного чата, работающий в режиме реального времени по протоколу **WebSocket**. 

##  Ссылки на проект

* **Frontend (GitHub Pages):** [👉 Открыть чат в браузере](https://milka79rus.github.io/ahj-chat/)
* **Backend (Amvera Cloud):** `https://ahj-backend-milka-milka79rus.amvera.io`
---

##  Реализованный функционал

* **Вход по псевдонимам:** При старте приложения открывается модальное окно авторизации. Проверка занятости никнейма происходит мгновенно через HTTP API сервера.
* **Списки пользователей (Real-time):** В левой панели динамически обновляется список всех участников, находящихся в данный момент в сети.
* **Интеллектуальное разделение сообщений:** * Собственные сообщения выравниваются по правому краю, а имя автора заменяется на системный маркер `You` (выделен красным цветом согласно спецификации).
  * Сообщения других участников выравниваются по левому краю с отображением их реальных никнеймов.
* **Контроль сессий:** Реализован автоматический выход из чата (событие `beforeunload`). При закрытии вкладки или перезагрузке страницы пользователь корректно удаляется со стороны сервера.
* **Таймстампы:** Каждое сообщение форматируется локализованной датой и временем (`ЧЧ:ММ ДД.ММ.ГГГГ`).

##  Технологический стек

* **Frontend:** JavaScript (ES6+), Fetch API (HTTP POST), WebSockets (`ws://` / `wss://`).
* **Сборщик:** Webpack 5 (сборка, минификация, обработка стилей и ресурсов).
* **Качество кода:** ESLint (конфигурация полностью валидирована, код очищен от ошибок).
* **CI/CD:** GitHub Actions (автоматический деплой сборки фронтенда на GitHub Pages при каждом пуше).
* **Backend:** Node.js (развернут на платформе Amvera Cloud с поддержкой постоянных безопасных WebSocket-соединений).

---

##  Архитектура (Environment Switching)

В коде реализовано автоматическое переключение окружения. Приложение самостоятельно определяет контекст выполнения:
* На `localhost` запросы идут к локальному серверу (`localhost:3000`).
* В продакшене приложение автоматически переключается на защищенные протоколы облачного сервера Amvera (`https://` и `wss://`), что полностью исключает ошибки блокировки смешанного контента (Mixed Content).
